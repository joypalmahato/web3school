# Web3School

**AI-powered Web3 career discovery and learning platform.**

Go from "I don't know what to learn" to "I have a marketable skill and I can prove it" — guided by AI every step of the way.

## Overview

Web3School helps users discover their ideal Web3 career path through an AI-powered conversation, then guides them through a personalized 90-day learning roadmap with daily lessons, an AI tutor, streak tracking, and a shareable Skill Passport.

### Key Features

- **AI Career Discovery** — Conversational quiz that matches users to 1 of 8 Web3 roles based on personality traits
- **Personalized Roadmap** — 12-week curriculum generated by Claude AI, tailored to the user's chosen role
- **Daily Learning Engine** — Bite-sized lessons, exercises, quizzes, and projects with AI-generated content
- **AI Tutor Chat** — Streaming AI tutor available on every lesson for real-time help
- **Gamification** — XP system, streaks, levels, progress dashboard, and calendar heatmap
- **Skill Passport** — Public portfolio page showcasing verified skills and completed projects
- **Role Explorer** — SEO-optimized public pages for all 8 Web3 career paths

## Getting Started

### Prerequisites

- Node.js 18+
- npm or yarn
- Supabase account
- Anthropic API key

### Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/joypalmahato/web3school.git
   cd web3school
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Configure environment variables**
   ```bash
   cp .env.example .env.local
   ```
   Fill in your API keys (see [Environment Variables](#environment-variables) below).

4. **Set up the database**
   - Create a Supabase project at [supabase.com](https://supabase.com)
   - Run the migration: paste `supabase/migrations/001_initial_schema.sql` in the SQL editor
   - Seed initial data: paste `supabase/seed.sql` in the SQL editor
   - Enable Google and GitHub OAuth in Authentication > Providers

5. **Start the dev server**
   ```bash
   npm run dev
   ```
   Open [http://localhost:3000](http://localhost:3000).

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | Supabase anonymous key |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | Supabase service role key (server-only) |
| `ANTHROPIC_API_KEY` | Yes | Anthropic API key for Claude |
| `RESEND_API_KEY` | No | Resend API key for transactional emails |
| `NEXT_PUBLIC_POSTHOG_KEY` | No | PostHog project token for analytics |
| `NEXT_PUBLIC_POSTHOG_HOST` | No | PostHog ingest host |
| `NEXT_PUBLIC_APP_URL` | Yes | App URL (e.g., `http://localhost:3000`) |
| `NEXT_PUBLIC_APP_NAME` | No | App display name (defaults to Web3School) |

## Deployment

### Vercel (Recommended)

1. Push your repo to GitHub
2. Import the project in [Vercel](https://vercel.com)
3. Add all environment variables from `.env.example` with production values
4. Set `NEXT_PUBLIC_APP_URL` to your production domain
5. Deploy — Vercel auto-detects Next.js and configures the build

### Supabase Production

1. Run `supabase/migrations/001_initial_schema.sql` on your production database
2. Run `supabase/seed.sql` to populate roles and skills
3. Configure OAuth providers (Google, GitHub) with production redirect URLs
4. Verify RLS policies are active on all tables
